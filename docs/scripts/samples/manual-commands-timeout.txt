Example run of escpos_status_probe.py. This is the evidence that the manual's
status commands do not answer over USB. The run below is with paper present,
but the paper-out run looks identical, which is the point.

device id (USB control readback works): MFG:HWASUNG;CMD:ESC/POS;MDL:HMK-072;CLS:PRINTER;DES:THERMAL PRINTER

The USB control readback (device id) works, so the printer is alive and
talking. It simply never answers the ESC/POS status commands over USB. The
paper-out run gives the same six timeouts, which is why the manual's commands
are useless for detecting paper here. Use read_paper_status.py instead.
